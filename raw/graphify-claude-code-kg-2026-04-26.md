## 概述

Graphify 是一款将代码库转化为可查询知识图谱的开源工具，可集成到 Claude Code、Codex 等 AI 编程助手中。本文从五层架构解析、与 Karpathy 工作流对比到六大真实场景实测，全面评测 Graphify 在代码精准定位、逻辑解释、论文与代码跨模态融合等方面的实际表现。

## 关键洞见（基于目录推断）

1. **Graphify vs 扁平 Wiki**：Karpathy 提出的扁平 wiki 模式存在"孤岛问题"——文档间无关系。Graphify 用知识图谱替代扁平文档，引入实体-关系结构，使 Claude Code 能进行图遍历式推理。

2. **AST + 语义双轨**：核心处理层同时运行 AST（语法树）解析和语义提取，前者捕获代码结构关系（调用图、依赖树），后者捕获语义相似性。两者协同比单独向量搜索更精准。

3. **跨模态是关键差异化**：论文节点与代码节点同图存储，支持"这篇论文的哪个公式对应代码的哪个函数"这类跨模态查询，这是纯 RAG 系统难以实现的。

4. **增量更新可持续**：update 命令支持增量更新图谱（无需重新全量分析），与 Wiki Compiler 的幂等防重原则一致。
