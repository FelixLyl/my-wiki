# 知识沉淀双轨机制

知识沉淀双轨机制是 Claude Code 中的"经验资产化"原则，将 AI 在项目中积累的经验按性质分为两条轨道分别存储。

## 两条轨道

### 私有轨 — Auto Memory（动态经验）

存储**个人化、环境相关**的经验：
- 本地环境配置（如"本地 Redis 端口是 6380"）
- 特定的调试技巧
- 项目特有的坑

**存储位置**：`.claude/memory/` 目录
**同步策略**：不提交 Git，属于开发者的私有笔记

### 共享轨 — Project Docs（通用智慧）

存储**具有普遍意义**的经验：
- 解决了通用 Bug 的方案
- 发现的更好架构模式
- 值得团队共享的最佳实践

**存储位置**：`CLAUDE.md` 的"常见问题"/"架构决策"章节，或 `docs/` 下的技术文档
**同步策略**：建议提交 Git，告知用户"已更新到 CLAUDE.md，建议提交给团队共享"

## 核心价值

| 问题 | 解决方式 |
|------|----------|
| 本地调试经验不污染团队 | 私有轨隔离 |
| 好经验无法沉淀到团队 | 共享轨 + Git 提交 |
| 项目越做越聪明 | 双轨持续积累 |

## 触发时机

任务结束时，AI 应主动评估：
1. 有没有值得写入 Auto Memory 的本地经验？
2. 有没有值得更新 CLAUDE.md 或文档的通用最佳实践？

这种主动沉淀的行为需要在 [[CLAUDE-md配置方法论]] 中显式声明，否则 AI 不会自动执行。

## 与知识库编译器的关系

[[个人知识库编译器模式]] 中的 [[LLM知识库三层架构]] 与双轨机制在精神上高度一致：Raw 层（不可变原始素材）对应私有轨的输入，Wiki 层（LLM 维护的知识库）对应共享轨的输出，Schema 层（配置文件）对应 CLAUDE.md 中显式声明的行为规范。两者都认为"分层隔离"是让 AI 产生持久价值的前提。
