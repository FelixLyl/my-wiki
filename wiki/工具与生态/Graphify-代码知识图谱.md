# Graphify：代码库知识图谱

Graphify 是一款将代码库转化为可查询知识图谱的开源工具，核心思路是用图结构替代扁平文档，使 [[Claude-Code源码架构|Claude Code]] 等 AI 编程助手能够进行关系式推理而非线性检索。

## 核心定位

Karpathy 提出的"扁平 wiki"模式适合个人知识管理，但用于代码理解时存在结构缺失问题：函数调用关系、模块依赖、论文概念与实现之间的映射，都无法用文档段落表达。Graphify 用**实体 + 关系**的图结构补上这一层，将代码库从"一堆文件"升级为"可查询的关系网络"。

## 五层架构

| 层次 | 功能 |
|------|------|
| 输入层 | 代码库扫描，支持多语言 |
| 核心处理层 | AST 解析 + 语义提取双轨并行 |
| 分析层 | 社区发现（Louvain 等算法），识别代码模块聚类 |
| 输出层 | 多格式导出，含 Obsidian 兼容格式 |
| 查询层 | 自然语言查询接口，供 Claude Code / Codex 调用 |

AST（语法树）捕获结构关系（调用图、依赖树），语义提取捕获意义相似性。两轨并行比单独向量搜索精准，也比单独 AST 分析更具语义弹性。这与 [[LLM-Wiki-Desktop]] 的"4信号知识图谱"设计理念相近。

## 与 Claude Code 原生记忆的区别

Claude Code 原生记忆系统（参见 [[Claude-Code源码架构]] 的五层记忆架构）是会话级和 CLAUDE.md 级的配置记忆，存储的是规则、偏好、历史工具观测值。Graphify 图谱引擎存储的是**代码结构本身的关系**，是静态的代码理解底座。两者互补：前者管行为约束，后者管知识检索。

## 六大实测场景

1. **代码精准定位**：查询"BM25 代码在哪个文件"，Graphify 通过图遍历给出精准路径，优于向量相似性搜索的模糊匹配
2. **复杂逻辑解释**：追踪函数调用链，解释多模块协同的实现逻辑
3. **参数值查询**：直接命中具体常量（如 Embedding 维度），无需 LLM 猜测
4. **增量更新**：`update` 命令支持合并 PR 后增量更新图谱，无需重新全量分析（与[[幂等防重模式]]一致）
5. **论文与代码跨模态融合**：导入 arXiv 论文后，图谱同时包含论文概念节点和代码实现节点，支持"这个公式在哪个函数里实现"的跨模态查询
6. **路径追踪**：`path` 命令显示从论文节点到代码节点的完整图谱路径

跨模态融合是 Graphify 最显著的差异化能力，纯 RAG 系统难以实现（向量相似度无法准确映射论文符号到代码变量名）。

## 高级功能

- **Obsidian 知识库生成**：导出为 Obsidian 格式，代码模块、函数、类自动成为笔记节点，双链表达调用关系
- **多 AI 助手兼容**：原生支持 Claude Code 和 Codex，具备通用 MCP 兼容接口

## 与 Karpathy 工作流的关系

Karpathy 的知识库工作流强调"把知识打平成 wiki"，Graphify 则主张"把代码打成图"——两者服务不同场景：
- Karpathy 模式适合**个人知识**（文章、笔记、洞见）
- Graphify 模式适合**代码库理解**（结构关系、调用图、跨模态映射）

两者可以组合：用 Graphify 生成代码理解图谱，导出为 Obsidian → 通过 [[个人知识库编译器模式]] 统一入库，实现代码知识与个人知识的融合。
