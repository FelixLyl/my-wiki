# 人格即防熵锚点——Agent 行为熵的第四维防线

[[Insight-Agent系统的三类熵增防线-2026-04-18]] 识别了三类熵：权限熵、流程熵、内容熵，对应三类防线：零信任、流程结构、幂等账本。但存在一个被遗漏的第四类熵——**行为熵**，而其防线恰好被 [[The-Agency-Agents]] 和 [[Andrej-Karpathy]] 从不同角度暗示出来。

## 行为熵的定义

Agent 在持续运行中，其行为风格会因上下文窗口重置、模型版本更新、Prompt 累积漂移而逐渐失稳：
- 同一任务在不同会话中产出风格不一致
- 与用户的协作模式因 Agent 记忆重置而反复摩擦
- "无状态 LLM" 的天然局限导致 Agent 每次重新解释自身角色

这是 [[Andrej-Karpathy]] 指出的核心问题的另一面：LLM "天然倾向于选一种解读静默执行"，在行为层面表现为 **每次会话都可能有细微不同的"自我认知"**。

## 人格文件作为行为防熵机制

[[The-Agency-Agents]] 项目的核心创见不仅是"做出很多专家 Agent"，而是以**人格文件**（identity + voice + workflow）作为 Agent 的行为锚点。人格不是装饰，它是让 Agent 跨会话保持行为一致的**外部记录系统**。

这与三类熵增防线的统一框架高度同构：

| 维度 | 熵的来源 | 防熵机制 | 核心原语 |
|------|---------|---------|---------|
| 安全 | 权限蔓延 | 零信任 + 操作可逆 | 人在回路 |
| 流程 | 上下文孤岛 | 流程结构化 | 强制接力 |
| 内容 | 重复处理漂移 | 幂等账本 | 增量优先 |
| **行为** | **风格 / 角色漂移** | **人格文件 + 约束配置** | **持续外化** |

四类防线的共同结构：**将 Agent 本来靠运气维持的一致性，改写为靠外部文件强制维持**。

## Karpathy 约束论的行为层延伸

Karpathy 的核心论断是：LLM 的价值不在于单次生成，而在于**通过持续约束产生可积累的产物**。他在知识库领域用 CLAUDE.md 约束行为；在编码领域用四准则约束隐藏假设。两种应用的底层结构相同：把"应该怎么做"外化为文件，而非依赖 LLM 的内化理解。

The Agency 的人格文件是同一逻辑在**协作层**的延伸：不是告诉 LLM "好好工作"，而是告诉它"你是谁、你怎么说话、你交付什么"——用具体的外部约束定义行为边界。

"I rarely touch the wiki directly. It's the domain of the LLM." — Andrej Karpathy

这句话同样适用于人格文件：*你不需要每次提醒 Agent 它是谁，人格文件会持续提醒它。*

## 推论

一个没有人格文件的 Agent 系统，会在足够多的会话后出现行为漂移：逐渐失去特定的协作风格，退化为"通用助手状态"。这是熵的自然方向。

人格文件的价值密度随使用时间增加——越是长期协作的 Agent，其人格锚点越是核心资产，而不只是入门配置。这与 wiki 知识库的复利逻辑完全一致：**持续写入的约束文件，是 Agent 系统对抗遗忘的基础设施**。
