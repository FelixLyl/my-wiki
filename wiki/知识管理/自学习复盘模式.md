# 自学习复盘模式

自学习复盘模式描述 AI Agent 从自身错误和纠正中持续积累经验、自动改进的行为模式。核心思想是：Agent 不应只是任务执行器，还应是自身经验的记录者和学习者。

## 两个实现层次

### Agent 层：self-improving-agent

自动记录纠正过的错误、踩过的坑，整理成**知识卡片**，供后续任务主动调用。

执行流程：
1. 任务执行中，用户纠正 Agent 的错误行为
2. Agent 将纠正内容结构化为知识卡片（错误描述 + 正确做法 + 适用场景）
3. 卡片写入记忆库（可对接 [[Agent记忆持久化]] 方案）
4. 后续相似任务时，Agent 主动检索相关卡片并应用

这是 [[知识沉淀双轨机制]] 的动态化版本：不等人工触发，Agent 在任务流程中自动识别值得记录的经验。

### Skill 层：self-evolving-skill

定期巡检已安装的 Skills，通过读取报错日志自动修改 Skill 代码以适配新环境。

适用场景：
- Skill 依赖的外部 API 接口发生变化
- 运行环境升级导致 Skill 出错
- 工具调用格式更新

这是 [[OpenClaw-Skill生态]] 的自维护机制，减少因环境变化导致的 Skill 失效。

## 与现有记忆机制的关系

| 机制 | 触发方式 | 存储对象 |
|------|---------|---------|
| 知识沉淀双轨机制 | 人工或任务结束后 | 工程经验（代码/架构） |
| 自学习复盘（Agent层） | 被纠正时自动 | 行为错误 + 修正方案 |
| 自学习复盘（Skill层） | 定时巡检 | Skill 代码 + 兼容性修复 |
| 个人知识库（wiki-compiler） | 入库触发 | 知识素材提炼 |

四种机制分别覆盖不同的记忆对象和触发时机，可组合构建完整的 Agent 记忆生态。

## 设计约束

- 自动修改代码的 Skill（self-evolving-skill）需要配合 [[OpenClaw-Agent安全体系]] 中的代码安全扫描，防止错误修复引入安全漏洞
- 知识卡片的可信度需要标注来源：人工纠正的卡片高可信，Agent 自推断的卡片低可信，避免"错误学习"
